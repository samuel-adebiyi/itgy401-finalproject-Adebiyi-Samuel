# SamTech Solutions Implementation

## What This Build Is

SamTech Solutions is a responsive, multi-page small IT consultancy website created for Adebiyi Samuel's ITGY401 final project. It demonstrates semantic HTML5, external CSS3, responsive layouts, structured data, accessible forms, and purposeful JavaScript interaction.

## What This Build Is Not

- It is not an online payment platform.
- It does not claim that sample clients or prices are real commercial records.
- It does not use a framework or backend; it must run directly on GitHub Pages.
- It does not ship placeholder pages, controls, or unfinished interactions.

## Folder Map

- `Source-of-truth/`: canonical product and privacy requirements.
- `Frontend-implementation/`: pages, components, styling, responsiveness, and interaction plan.
- `Backend-implementation/`: static-site boundaries and form/data contracts.
- `Feature-Pitch-For-Investors/`: business case and positioning.
- `report-findings/`: research, risks, edge cases, and grading coverage.

## Governance Rules

1. The source of truth wins when code and documentation disagree.
2. A task is checked only after implementation and verification.
3. Every visible control must perform a real action.
4. Content must be honest; invented demonstrations must be labelled.
5. Accessibility, mobile behavior, and keyboard use are requirements, not polish.
6. Amend the source of truth deliberately before changing agreed scope.

## Core Product Thesis

A small organisation should understand what SamTech Solutions offers, see transparent service options, and send a clear project enquiry without navigating a complicated website.
